# 에디토리얼 가이드

- **상태:** Draft
- **날짜:** 2026-06-15
- **참조:** `docs/content-model.md`(필드), `docs/authoring-workflow.md`(절차), `docs/accessibility-checklist.md`, `docs/seo-spec.md`

글을 **어떻게 쓰는가**의 기준. *무엇을·어디에*(파일·frontmatter·라우트)는 다른 문서가 정하고,
이 문서는 목소리·문체·구성 같은 글 품질을 일정하게 유지한다.

## 1. 목적과 독자

- 이 블로그는 **개발하며 배운 것을 기록**하는 노트다("배운 것들").
- 독자는 같은 문제를 마주한 개발자. 글을 읽고 **바로 적용할 한 가지**를 얻어가게 한다.
- 모든 톤·구성 결정은 "독자가 이해하고 써먹을 수 있는가"를 기준으로 한다.

## 2. 보이스 & 톤

- **존댓말, 1인칭.** "~합니다 / ~했습니다." 일관 유지. 반말·구어체 혼용 금지.
- **담백하고 정확하게.** 과장·낚시·단정(특히 미검증 내용)을 피한다. 모르는 건 모른다고 쓴다.
- **경험 중심.** "정답"을 가르치기보다 "내가 이렇게 했고 이렇게 배웠다"를 공유한다.
- 한 줄 정체성: *친근하지만 정확하게.*

## 3. 문체 · 표기 규칙

- 능동태 우선, 한 문장 한 메시지. 긴 문장은 끊는다.
- 외래어는 한글 표기를 기본으로 하되, 코드 식별자·고유 API명은 원어 + 백틱.
- 숫자·단위는 아라비아 숫자(`3개`, `200ms`), 날짜는 `YYYY-MM-DD`.
- 맞춤법·띄어쓰기는 국립국어원 기준. 발행 전 1회 검수.

### 용어 표기 (자주 흔들리는 것)

| 권장 | 지양 | 비고 |
|------|------|------|
| 컴포넌트 | 콤포넌트 | 한글 표기 |
| 렌더링 | 랜더링 | |
| `props`, `state`, `API` | 프롭스, 스테이트 | 코드 개념은 원어+백틱 |
| 서버 컴포넌트 | 서버컴포넌트 | 띄어쓰기 |
| 자바스크립트 / `JavaScript` | 자바 스크립트 | 문맥 따라 한글/원어 |

> 용어는 한 글 안에서 한 가지로 통일한다(한글↔원어 섞지 않기).

## 4. 글 구조

- **제목:** 핵심 키워드를 앞에, 길이는 간결하게. 낚시·과장 금지. (SEO 타이틀과 연동 — `seo-spec.md`)
- **도입:** "왜 이 글인가 / 어떤 문제를 푸는가"를 2~3문장으로 먼저.
- **본문:** 헤딩은 `##`(h2)부터. `#`(h1)은 제목이 차지한다. 레벨 건너뛰지 않기.
- **마무리:** 핵심 요약 또는 다음 단계 한 줄.
- **시리즈:** 연재는 frontmatter `series`로 묶고, 도입에 "N편 중 M편"을 밝힌다.

## 5. 코드 · 예제

- 코드 펜스에 **언어 명시**(```` ```ts ````) — 하이라이팅·복사 동작.
- **최소 실행 가능 예제.** 핵심만 남기고 곁가지 제거. 너무 길면 요점만 발췌.
- 주석은 **WHY만**(무엇을 하는지 설명하는 주석 금지 — 레포 코드 가드레일과 동일 철학).
- 본문 속 식별자·파일명·명령어는 백틱(`useState`, `npm run dev`).

## 6. 이미지 · 인용 · 링크

- **alt 필수.** 의미 있는 이미지엔 설명적 alt, 장식은 `alt=""`(접근성 체크리스트와 연동).
- 스크린샷·다이어그램엔 필요 시 캡션으로 맥락 보강.
- **출처·저작권:** 외부 이미지·인용은 출처 명시. 인용은 짧게, 원문 그대로 표기.
- 링크 텍스트는 목적이 드러나게("여기" 금지). 외부 링크 출처를 신뢰할 수 있게.

## 7. 태그 · 요약 작성

- **태그(`tags`):** 소문자 영문 권장, 글당 2~4개. 너무 세분화하지 않기(재사용 가능한 단위). (스키마는 `content-model.md`)
- **요약(`description`):** 70~160자, 글의 핵심을 한 문장으로. 목록 발췌·검색 스니펫·OG에 그대로 쓰인다(`seo-spec.md`).

## 8. 민감 사항

- 회사 기밀·미공개 정보·타인의 개인정보를 쓰지 않는다.
- 미완성·미검증 내용을 단정적으로 쓰지 않는다("아마 / 확인 필요"를 명시).
- 특정 인물·조직을 부정확하게 단정·비방하지 않는다.
