# 정보 구조 (IA)

- **상태:** Draft
- **날짜:** 2026-06-15
- **참조:** 와이어프레임 `docs/wireframes/`, 로드맵 `docs/roadmap/`

화면을 **기능 단위**로 정의하는 기준 문서다. 구현 명칭(브랜드·페이지 별칭·라이브러리)이 아니라
"사용자에게 어떤 기능을 제공하는가"로 구분한다. 구체적인 라우트·콘텐츠명은 맨 아래 매핑 표에서만 다룬다.

## 기능 영역

사이트는 여섯 가지 기능 영역으로 구성된다.

### 1. 진입 (Landing)

사이트의 첫 화면. 정체성을 짧게 드러내고 주요 기능으로 보내는 진입점.

- **슬림 인트로** — 아바타 + 한 줄 소개 (거창한 히어로 카피 없음. 연결 정보는 푸터)
- **최근 글** 목록 — 블로그 1차 콘텐츠 동선
- (확장) 연도 아카이브

### 2. 콘텐츠 탐색 (Browse)

읽을 글을 찾는 기능. 두 가지 탐색 축을 제공한다.

- **목록 + 요약:** 제목·작성일·주제·발췌
- **주제 기반 필터:** 주제 태그로 좁히기
- **시간 기반 탐색:** 연도 등 작성 시점으로 묶어 보기
- (향후) 페이지 분할 — 목록 임계치 초과 시 (ADR-012)

> 태그(`/tag`)·아카이브(`/archive`)는 **글 목록(`/writing`)의 필터된 하위 뷰**다. 글 목록에서 진입하며,
> 복귀 동선은 **"← 모든 글"**(글 목록)으로 통일한다. 글 항목은 세 화면 모두 동일한 `PostCard`를 쓴다.

### 3. 콘텐츠 소비 (Read)

한 편의 글을 읽는 기능과 읽기 보조.

- 본문 렌더링(코드·이미지 포함)
- **읽기 보조:** 문서 내 목차, 이전/다음 글 이동
- **반응:** 댓글 등 피드백 채널
- 메타 정보: 작성일·주제 태그

### 4. 감상 기록 (Collection)

소비한 미디어를 모아 보여주는 큐레이션 기능.

- **유형별 분류:** 도서 · 애니 카테고리 전환
- 항목 카드: 표지 + 제목 (별점 없음 — 텍스트 기록으로 대체)
- **항목 상세:** 항목 클릭 시 상세 페이지(`/library/[slug]`)로 이동. 상세에서 **문장수집**(책·작품에서 밑줄 친 인용)과 **서평**(감상 기록)을 표출한다
- (연계) 진입 화면의 하이라이트로 노출 가능

### 5. 메모 (Notes)

게시글만큼 길지 않아도 되는 짧은 기록. 기술 단상·아이디어·관심사를 가볍게 남긴다.

- 다양한 형태: 한 줄 메모 · 체크리스트 · 코드 스니펫 · 이미지 · 태그 모음
- **주제 기반 필터** + 검색

### 6. 저자/사이트 정보 (About)

운영자·사이트를 소개하는 기능.

- 프로필: 사진 + 상태 + 한 줄 소개
- **연락 채널:** 외부 프로필·이메일 등

## 기능 간 관계

```
 진입(Landing) ──┬─→ 콘텐츠 탐색(Browse) ──→ 콘텐츠 소비(Read)
                ├─→ 메모(Notes)
                ├─→ 감상 기록(Collection)
                └─→ 저자 정보(About)
```

전역 내비게이션은 기능 영역을 직접 노출한다(순서): **글 · 메모 · 책장 · 소개**.
진입은 브랜드/로고로 접근한다. 공유 헤더·푸터의 와이어프레임 정본은 `docs/wireframes/layout.html`.

## 검색 (컨텍스트 스코프)

전역 헤더에서 **모달**로 호출하되, 검색 범위는 **현재 섹션에 한정**한다. 글 화면에선 글만, 메모 화면에선 메모만, 책장에선 책장만 검색한다 — 콘텐츠 유형이 섞이지 않아 결과가 맥락에 맞다.

| 항목 | 정의 |
|------|------|
| 스코프 규칙 | 글 영역(`/writing`·`/tag`·`/archive`) → 글만 · 메모(`/notes`) → 메모만 · 책장(`/library`) → 책장만 · 중립 화면(홈·소개) → 글 기본 |
| 인덱스 구조 | **타입별 분리 인덱스**(writing / notes / library), 빌드 시 생성 |
| 대상 필드 | 글·메모 → 제목 · 요약(발췌) · 주제 태그 / 책장 → **제목만** · 공통적으로 **본문 제외**(페이로드, ADR-004) |
| 스코프 전환 | 모달 내 스코프 전환 탭은 향후 검토 — 기본은 컨텍스트 한정 |
| 빈 결과 | 0건 시 안내 + 해당 섹션 전체 보기 동선 — 상태 정책 참조 |

## 분류 체계 (Taxonomy)

기능을 받치는 분류 축은 네 가지다. 모두 콘텐츠와 같은 곳에서 관리한다(별도 CMS 없음).

| 축 | 적용 기능 | 설명 |
|----|-----------|------|
| 주제 태그 | 탐색 · 소비 · 메모 | 글·메모의 1차 분류, 필터 |
| 작성 시점 | 탐색 | 연도 등 시간 기반 묶음 |
| 미디어 유형 | 감상 기록 | 도서·애니 카테고리 |
| 시리즈 | 소비 | 연재 글 묶음·순서 (시리즈 네비) |

> **연관 글**은 별도 축이 아니라 주제 태그 겹침으로 파생되는 관계다.

각 분류축이 실제 어떤 frontmatter 필드로 구현되는지는 `docs/content-model.md`의 "분류축 ↔ 필드 매핑" 참조.

## 구현 매핑 (참고)

기능 영역을 현재 라우트·와이어프레임에 연결한 참고 표. **이 표만 구현 명칭을 다룬다.**

> **동기화 책임:** 이 표는 정본이 아니라 안내도다. 라우트의 정본은 `src/app/` 디렉터리 구조이며,
> 라우트가 바뀌면 `docs-manager`가 `src/app/`을 읽어 이 표를 대조·갱신한다(별도 스크립트를 두지 않는다).
> 함께 점검할 대상: `src/app/`(정본) · 이 표 · `next.config`(리다이렉트) · `docs/roadmap/m1-blog-core.md`.

| 기능 영역 | 라우트 | 와이어프레임 |
|-----------|--------|--------------|
| 진입 | `/` | `home.html` |
| 콘텐츠 탐색 | `/writing` | `list.html` |
| 콘텐츠 소비 | `/writing/[slug]` | `detail.html` |
| 시간 기반 탐색 (인덱스) | `/archive` | `archive-index.html` |
| 시간 기반 탐색 | `/archive/[year]` | `archive.html` |
| 주제 탐색(태그) | `/tag/[tag]` | `tag.html` |
| 메모 | `/notes` | `notes.html` |
| 감상 기록 (책장) | `/library` | `library.html` |
| 감상 기록 상세 | `/library/[slug]` | (와이어프레임 없음) |
| 저자 정보 | `/about` | `about.html` |
| (공유 셸) | — | `layout.html` |
| (반응형 변형) | — | `mobile.html` |
| (상태: 빈·에러·로딩) | — | `states.html` |
| (컴포넌트 시트, 시각 정본) | — | `components.html` |

> 하단 4개 행(`—` 라우트)은 특정 라우트에 대응하지 않는 **횡단 와이어프레임**이다.
> 공유 셸·반응형·상태·컴포넌트 시트는 모든 기능 영역에 걸쳐 적용된다.

### 유틸리티 엔드포인트

화면이 아니라 머신·크롤러용 출력. 와이어프레임이 없고 App Router 파일 컨벤션으로 생성한다.

| 엔드포인트 | 목적 | App Router 구현 | 출처 |
|-----------|------|----------------|------|
| `/sitemap.xml` | 사이트맵 | `app/sitemap.ts` | M3 |
| `/robots.txt` | 크롤러 규칙 | `app/robots.ts` | M3 |
| `/feed.xml` | RSS 피드 | `app/feed.xml/route.ts` | M3 |
| 동적 OG 이미지 | SNS 미리보기 | 세그먼트별 `opengraph-image.tsx` | M3 |
| 404 | 미정의 경로 | `app/not-found.tsx` | M1 |

URL 규칙: 소문자 케밥케이스, 후행 슬래시 없음, 한글 slug 금지. 라우트 변경 시 구→신 301 리다이렉트를 함께 정의한다.

### 페이지네이션

**현재는 목록을 분할하지 않는다.** 글 수가 적어 목록(`/writing` 등)은 전체를 한 페이지에 노출하며, 찾기는 태그·아카이브·검색이 담당한다. 개별 글 URL `/writing/[slug]`은 이와 무관하게 슬러그 기반으로 접근한다.

목록 한 페이지 로딩이 무거워지는 시점(대략 글 30~50편 초과)에 **경로 세그먼트** 방식 `{목록}/page/{n}`(예: `/writing/page/2`)으로 페이지네이션을 도입한다. 방식·전환 기준의 배경은 ADR-012 참조.

## 렌더링 전략

페이지별 렌더링 방식. Next.js 16 App Router 기준.

| 라우트 | 전략 | 근거 |
|--------|------|------|
| `/` | SSG | 정적 콘텐츠, 빌드 시 확정 |
| `/writing` | SSG | 빌드 시 글 목록 생성 |
| `/writing/[slug]` | SSG + `generateStaticParams` | MDX 파일 기반 정적 생성 |
| `/tag/[tag]` | SSG + `generateStaticParams` | 태그 목록 빌드 시 확정 |
| `/archive` | SSG | 연도 폴더 목록 빌드 시 확정 |
| `/archive/[year]` | SSG + `generateStaticParams` | 연도 목록 빌드 시 확정 |
| `/notes` | SSG | 정적 노트 데이터 |
| `/library` | Dynamic | `?type=` searchParams로 카테고리(anime·book) 분기 |
| `/library/[slug]` | SSG + `generateStaticParams` | MDX 파일 기반 정적 생성. `dynamicParams = false` — 미존재·draft는 404 |
| `/about` | SSG | 정적 콘텐츠 |

새 글은 빌드·배포를 통해 반영된다. 실시간 갱신이 필요한 경우 ISR(`revalidate`) 도입을 검토한다.

## 상태 정책 (Empty / Error / Not Found)

비정상·경계 상황의 IA 차원 동작 규칙. 시각 정본은 `docs/wireframes/states.html`.

| 상태 | 발생 조건 | 동작 |
|------|----------|------|
| 404 (Not Found) | 미정의 슬러그·경로 (`/writing/없는글`, 존재하지 않는 태그·연도) | 전역 `not-found` — 안내 + 홈·글 목록 복귀 동선 |
| 빈 목록 (Empty) | 태그·아카이브·검색 결과 0건 | 빈 영역 대신 안내 메시지 + 대체 동선(전체 글 보기) |
| 에러 (Error) | 렌더·데이터 로드 실패 | `error` 바운더리 — 재시도 동선, 전역 셸은 유지 |
| 로딩 (Loading) | SSG라 페이지 전환엔 거의 없음 | 클라이언트 상호작용(검색 모달 등)에 한해 로딩 표시 |

## 전역 정책 (Global)

### 언어 / 로케일

단일 로케일 **`ko`**. 다국어(i18n)는 비대상 — `<html lang="ko">` 고정, 로케일 라우팅(`/en/...`) 없음.

### 브레드크럼 (Breadcrumb)

사이트 계층이 얕아(최대 2뎁스) 기본적으로 브레드크럼을 두지 않는다. 현재 위치는 전역 내비 활성 상태 + 페이지 제목으로 표시한다.

**2뎁스 섹션 상세 뷰는 브레드크럼을 허용한다.** 섹션별 복귀 패턴은 다음과 같이 다를 수 있다.

| 섹션 | 상세 복귀 패턴 |
|------|--------------|
| 글(`/writing/[slug]`) | 텍스트 복귀 링크 "← 모든 글" (본문 상·하단) |
| 태그·아카이브 | 텍스트 링크 "← 모든 글" (상단) |
| 책장(`/library/[slug]`) | 브레드크럼(홈 / 책장 / 제목) + 하단 "← 책장으로" 복귀 동선 |

향후 섹션이 늘어나면 복귀 패턴을 통일하는 방향을 검토한다.

### 콘텐츠 가시성

| 상태 | 노출 범위 |
|------|----------|
| `published` | 전 영역(목록·태그·아카이브·검색·사이트맵·RSS) 노출 |
| `draft` | **dev 환경 전용** — 프로덕션 빌드·사이트맵·RSS·검색에서 제외 (M1 draft 모드) |

가시성을 판단하는 frontmatter 필드 정의는 `docs/content-model.md` 참조.
