## Milestone Docs (`mN-*.md`)

새 마일스톤 문서는 `MILESTONE_TEMPLATE.md`를 복제해서 만든다. 섹션 구조는 템플릿대로 고정하며 추가 섹션을 만들지 않는다.

**작성 규칙:**

- **진행 상태:** `예정` | `진행 중` | `완료 (YYYY-MM-DD, vX.Y.Z)` — 완료일은 CHANGELOG 릴리즈일과 일치시킨다
- **Blocked by:** 실제로 막는 의존성만 적는다 ("완료 권장" 같은 비의존성 금지)
- **DoD:** 체크박스로 쓰고 완료 시 체크한다. 가능하면 측정 가능한 기준(지표·테스트)으로
- **태스크 보류:** 취소선 + `보류 → backlog 이월`로 표기하고, 반드시 `backlog.md`에 항목을 추가한다
- **스코프 변경:** 마일스톤 진행 중 범위가 바뀌면(항목 제거·보류·정책 확정) `## 변경 이력` 표에 `날짜 | 변경 | 사유` 한 줄을 남긴다

## Constraints

| Constraint | 규칙 | 이유 |
|-----------|------|------|
| **템플릿 밖 섹션** | "Technical Risks", "향후 검토 방향", "스코프 확정" 등 추가 섹션 금지. 리스크 분석은 구현 계획(planner) 단계, 미확정 아이디어는 `backlog.md`에서 | 마일스톤 문서는 범위·완료 기준 추적용 — 논의·분석이 섞이면 문서가 낡는다 |
| **인라인 이슈 참조** | Tasks·DoD 등 본문에 이슈 번호/링크 금지. Related Issues / Links에 `- #NN — 설명` 형태로 한 번만 (순수 `#NN` 텍스트, GitHub URL 링크 금지) | 같은 번호 반복 방지, 이슈 참조 위치 단일화 |
