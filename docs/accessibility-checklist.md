# 접근성 체크리스트

- **상태:** Draft
- **날짜:** 2026-06-15
- **기준:** WCAG 2.2 AA
- **참조:** `DESIGN.md`(토큰), `docs/information-architecture.md`(화면), `docs/wireframes/`

화면 구현·리뷰 시 점검하는 접근성 항목. 기능 영역(진입·탐색·소비·감상·저자)과 상태 화면 전반에 적용한다.
각 항목은 PR에서 체크하며, 해당 토큰/화면을 함께 표기했다.

## 1. 구조 · 시맨틱

- [ ] `<html lang="ko">` 설정
- [ ] 랜드마크 사용: `header` / `nav` / `main` / `aside` / `footer` 각 1개 역할 명확
- [ ] 페이지당 `h1` 하나, 헤딩 레벨 건너뜀 없음 (소비 화면 본문 포함)
- [ ] 본문 시작으로 가는 "본문 바로가기(skip link)" 제공
- [ ] 목록·책장 그리드는 시맨틱 리스트(`ul`/`li`)로 마크업
- [ ] 글 상세는 `article`, 목차는 `nav aria-label="목차"`

## 2. 키보드

- [ ] 모든 인터랙티브 요소(링크·버튼·탭·칩·페이지네이션)를 Tab으로 도달·조작 가능
- [ ] 포커스 표시가 항상 보임 — `{colors.primary-focus}`(#0071e3) 2px outline 사용, `outline:none` 단독 금지
- [ ] 포커스 순서가 시각적 순서와 일치
- [ ] 검색 모달(m3): 열면 포커스 이동, 내부 포커스 트랩, Esc로 닫고 트리거로 복귀
- [ ] 키보드 트랩 없음 (모달 외 영역)
- [ ] 책장 카테고리 탭: 화살표 키 이동 + `role="tablist"` 패턴

## 3. 대비 · 색

- [ ] 본문 텍스트 대비 ≥ 4.5:1, 큰 텍스트(≥24px 또는 19px Bold) ≥ 3:1
- [ ] 라이트/다크 모드(m1) **양쪽 모두** 대비 충족
- [ ] 정보를 색만으로 전달하지 않음 — 링크는 밑줄/문맥 병행, 별점은 텍스트 값 동반
- [ ] 비활성/플레이스홀더 텍스트도 최소 대비 확인 (`{colors.ink-muted-48}`는 본문 금지, 캡션·법적 고지 한정)
- [ ] 포커스 링·경계선이 배경과 ≥ 3:1 (UI 컴포넌트 대비)

## 4. 이미지 · 미디어

- [ ] 의미 있는 이미지(작품 표지·포스터·OG)에는 설명적 `alt`
- [ ] 장식 이미지는 `alt=""`로 보조기술에서 제외
- [ ] 코드 블록: 스크린리더가 읽을 수 있는 텍스트, 가로 스크롤 시 키보드 접근 가능
- [ ] 코드 복사 버튼: 접근 가능한 이름(`aria-label="코드 복사"`) + 복사 완료 피드백 전달

## 5. 폼 · 인터랙션

- [ ] 검색 입력에 연결된 `label`(시각 숨김 가능) 제공
- [ ] 입력 오류 상태를 텍스트로 안내(색만으로 X), `aria-invalid`/`aria-describedby`
- [ ] 버튼은 `<button>`, 링크는 `<a>` — 역할에 맞게 사용
- [ ] 댓글(giscus, m3): iframe `title` 지정, 키보드 도달 가능

## 6. 모션

- [ ] `prefers-reduced-motion: reduce` 존중 — 스켈레톤 시머·`scale(0.95)` 프레스 등 애니메이션 축소/제거
- [ ] 자동 재생·무한 루프 애니메이션은 정지 수단 제공 또는 reduce에서 비활성

## 7. 상태 화면 (빈 / 로딩 / 에러)

- [ ] 로딩: 스켈레톤에 `aria-busy="true"` 또는 "불러오는 중" 라이브 안내
- [ ] 빈 화면: 안내 문구 + 다음 행동(예: "첫 글 쓰기")을 텍스트로 제공
- [ ] 에러: `role="alert"`/`aria-live="assertive"`로 실패 안내, 재시도 버튼 포커스 가능
- [ ] 404: `h1`로 명확히 표기 + 목록 복귀 링크

## 8. 반응형 · 터치

- [ ] 터치 타깃 최소 44 × 44px (`DESIGN.md` Touch Targets)
- [ ] 320px 폭에서 가로 스크롤 없이 reflow (모바일 단일 컬럼)
- [ ] 200% 확대 시 콘텐츠 손실·겹침 없음
- [ ] 모바일 햄버거 메뉴: 펼침/접힘 상태 `aria-expanded` 반영
- [ ] 하단 탭바: 현재 위치 `aria-current="page"`

## 동기화 책임

새 컴포넌트·상태·인터랙션을 추가하면 `docs-manager`가 이 체크리스트에 해당 항목이 있는지 확인하고,
없으면 추가한다. 디자인 토큰(포커스·대비) 변경 시 3·2장을 함께 갱신한다.
