# PRD: 개인 블로그

## 제품 개요

개발, 일상 등 배우고 경험한 것들을 자유롭게 기록하는 개인 블로그.
글을 쓰고 싶을 때 빠르게 발행할 수 있고, 독자가 원하는 글을 쉽게 찾을 수 있는 것이 핵심이다.

## 목표

| # | 목표 | 측정 기준 |
|---|------|---------|
| 1 | 콘텐츠 발행이 쉬워야 한다 | MDX 파일 추가만으로 글이 발행된다 |
| 2 | 독자가 원하는 글을 찾을 수 있어야 한다 | 태그·검색·아카이브로 3회 이내 탐색 도달 |
| 3 | 글 읽기에 집중할 수 있어야 한다 | 광고 없음, 팝업 없음, 본문 최대 너비 제한으로 가독성 확보 |
| 4 | 검색 엔진에 노출되어야 한다 | 주요 페이지가 Google 색인에 포함된다 |

## 사용자 시나리오

**시나리오 A — 글 작성자 (나)**

업무나 학습 중 기록해두고 싶은 내용이 생겼을 때, 별도 CMS나 관리 UI 없이 MDX 파일을 추가하는 것만으로 글을 발행하고 싶다. 아직 완성되지 않은 초안은 `draft: true`로 표시해두고 로컬에서만 확인하다가, 완성되면 플래그를 제거하여 발행한다. 배포는 파일 push만으로 자동 완료된다.

- 맥락: 새로 배운 개념, 삽질 경험, 일상적인 생각을 빠르게 기록하고 싶다
- 흐름: MDX 파일 생성 → 로컬에서 미리보기 → 이상 없으면 push → 자동 배포
- 기대: 글 발행에 5분 이상 걸리지 않는다

---

**시나리오 B — 처음 방문한 독자**

검색 엔진에서 특정 문제의 해결법을 찾다가 블로그 글에 진입한다. 처음 보는 블로그이기 때문에 이 글이 믿을 만한지 빠르게 판단하고, 필요한 내용을 효율적으로 읽고 싶다. 글을 다 읽은 후 관련 주제의 다른 글이 있다면 자연스럽게 탐색하게 된다.

- 맥락: 기술 문제 해결, 개념 학습, 혹은 일상적인 주제로 검색 경유 진입
- 흐름: 글 진입 → 제목·날짜·읽기 시간으로 신뢰도 확인 → 목차로 필요한 섹션 이동 → 본문 읽기 → 연관 글 또는 태그로 추가 탐색
- 기대: 원하는 정보를 빠르게 찾고, 블로그에 머무를 이유가 생긴다

---

**시나리오 C — 재방문 독자**

이전에 유용한 글을 읽은 적 있어 다시 방문한다. 새로 올라온 글이 있는지 확인하거나, 기억이 흐릿한 이전 글을 다시 찾으려 한다. 태그나 검색으로 원하는 글을 빠르게 찾을 수 있고, 글을 읽으면서 댓글로 의견을 남기기도 한다.

- 맥락: 최신 글 확인 또는 이전에 읽은 글 재탐색
- 흐름: 목록 페이지에서 최신 글 확인 → 태그 필터 또는 검색으로 원하는 글 탐색 → 글 읽기 → 필요 시 댓글 작성
- 기대: 이전 방문 이후 새로운 글을 쉽게 발견하고, 원하는 글을 3번 이내 클릭으로 찾는다

## 기능 요구사항

### 콘텐츠

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 글 목록 | 최신순 정렬, 제목·날짜·태그·요약 표시 | 상 |
| 글 상세 | 본문 렌더링, 코드 신택스 하이라이팅, 날짜·읽기 시간 표시 | 상 |
| 초안 모드 | 초안 글은 로컬 개발 환경에서만 보인다 | 상 |

### 탐색

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 태그 필터 | 목록 페이지에서 태그로 글을 필터링한다 | 상 |
| 태그 페이지 | 태그별 글 목록 페이지 (`/tag/[tag]`) | 중 |
| 연도별 아카이브 | 연도별 글 목록 페이지 (`/archive/[year]`) | 중 |
| 검색 | 제목·본문 기반 퍼지 검색, 모달로 제공 | 중 |

### 읽기 경험

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 목차(TOC) | 헤딩 기반 목차, 데스크톱 사이드바에 고정 표시 | 상 |
| 시리즈 네비게이션 | 연관 글을 순서대로 묶어 이전/다음 편 탐색 가능 | 중 |
| 이전/다음 글 | 날짜 기준 인접 글 이동 | 중 |
| 연관 글 | 태그 겹침 기준 최대 3개 추천 | 중 |
| 공유 | URL 복사 | 하 |
| 댓글 | giscus (GitHub Discussions 기반) | 하 |

### 레이아웃 / 접근성

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 다크/라이트 모드 | 시스템 설정 기본값, 수동 전환 가능 | 상 |
| 반응형 | 모바일·태블릿·데스크톱 전 breakpoint 대응. 모바일에서 사이드바·TOC 숨김, 터치 친화적 탭 간격, 가로 스크롤 없음 | 상 |
| 네비게이션 | 최상단 고정 헤더, 하단 푸터 | 상 |

### SEO

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 페이지별 메타데이터 | 글 제목·요약이 `<title>`, `description`에 반영 | 중 |
| OG 이미지 | 글별 동적 생성 | 중 |
| 사이트맵 | 자동 생성 | 중 |

## 비기능 요구사항

- **성능:** 글 목록 페이지 LCP 2.5초 이내 (정적 생성 기준)
- **접근성:** 키보드 탐색 가능, 이미지 alt 텍스트 필수
- **유지보수:** 글 추가 시 코드 변경 없이 파일만 추가하면 된다

## Out of Scope

- 회원가입 / 로그인
- 관리자 대시보드
- 뉴스레터 구독
- 다국어 지원
- 페이지네이션 (글 전체를 한 페이지에 표시, 필요 시 재검토)
